Lists: Ordered & Indexed

  • Sequential storage in memory
  • Access by index: O(1)
  • Search by value: O(n)
  • Good for: ordered sequences, iteration
  • Bad for: frequent lookups, existence checks
© mui-group
4 / 15

Dictionaries: Key-Value Pairs

  • Hash-based direct access
  • Lookup by key: O(1)
  • Insert/Delete: O(1)
  • Good for: mappings, fast lookups, caching
  • Critical for: tokenizers, embeddings, indexes

Mutable vs Immutable

Immutable

  • Strings, tuples, numbers
  • Cannot change in-place
  • Operations create new objects
  • Different memory address
  • Safe for concurrent access

Mutable

  • Lists, dicts, sets
  • Can modify in-place
  • Same object, different content
  • Same memory address
  • Risk of side effects
© mui-group

Memory References with id()

# Immutable: String creates new object
original_str = "hello"
print(id(original_str))  # 140234567890

modified_str = original_str.upper()
print(id(modified_str))  # 140234567999 ← Different!

# Mutable: List modifies in-place
original_list = [1, 2, 3]
print(id(original_list))  # 140234568100

original_list.append(4)
print(id(original_list))  # 140234568100 ← Same!

Understanding memory behavior prevents bugs in data pipelines!
