Part 1 · Element-wise

Operating on whole arrays at once

Element-wise operations apply independently to each matching element of two arrays. Addition, multiplication, division, and powers all happen with no loop. NumPy runs them in optimized C code.

import numpy as np

arr1 = np.array([1, 2, 3, 4])
arr2 = np.array([10, 20, 30, 40])

# Each result element combines the matching pair
print(arr1 + arr2)   # [11 22 33 44]
print(arr1 * arr2)   # [10 40 90 160]
print(arr2 / arr1)   # [10. 10. 10. 10.]
print(arr1 ** 2)     # [ 1  4  9 16]
The core idea

No loops needed. NumPy pushes the work into compiled C, which is where the speed of every deep learning framework comes from.

Part 2 · Scalar broadcasting

One value stretched across an array

Combine a single scalar with an array and NumPy conceptually repeats that scalar to match the array's shape, then does the element-wise operation. Nothing is actually copied in memory.

scalar = 5
array = np.array([1, 2, 3, 4])

print(scalar * array)   # [ 5 10 15 20]
print(array + scalar)   # [6 7 8 9]

# Internally NumPy treats 5 as if
# it were [5, 5, 5, 5], then does
# the element-wise operation
Where you will see it

Adding a bias term to every neuron output is exactly this pattern, a scalar (or vector) broadcast across many values.

Part 3 · The dot product

The sum of element-wise products

The dot product multiplies two vectors element by element and adds up the results into a single number. It is the mathematical core of a neuron's weighted sum.

vec1 = np.array([1, 2, 3])
vec2 = np.array([4, 5, 6])

# (1x4)+(2x5)+(3x6) = 4+10+18 = 32
print(np.dot(vec1, vec2))   # 32
print(vec1 @ vec2)          # 32 (same)
Why it matters

A neuron's weighted sum w · x is a dot product, so this one operation is the heartbeat of every neural network.

Part 4 · Matrix multiplication

Many dot products at once

Matrix multiplication computes a dot product between every row of the first matrix and every column of the second. The inner dimensions must match: an (m, n) matrix times an (n, p) matrix gives an (m, p) result.

mat_a = np.array([[1, 2, 3],
                  [4, 5, 6]])   # Shape: (2, 3)
mat_b = np.array([[10, 11], [20, 21], [30, 31]])   # Shape: (3, 2)

result = mat_a @ mat_b           # Shape: (2, 2)
print(mat_a[0] @ mat_b[:, 0])    # result[0, 0] as a dot product

Part 4 · Matrix multiplication

The most common NumPy bug

Two operators look almost the same but do completely different things.

A * B

  • Element-wise product
  • Multiplies matching positions
  • Shapes must match (or broadcast)
  • Result has the same shape

A @ B

  • Matrix multiplication
  • Row-by-column dot products
  • Inner dimensions must match
  • Also np.dot or np.matmul
Watch out

A * B is element-wise, while A @ B is matrix multiplication. Mixing them up is one of the most common NumPy bugs. The modern, preferred style for matrix multiply is the @ operator.

Part 5 · Broadcasting rules

Three rules that align any shapes

  • Rule 1: if ranks differ, prepend 1s to the smaller shape
  • Rule 2: dimensions are compatible when they are equal, or one of them is 1
  • Rule 3: the result takes the maximum size along each dimension
arr_2d = np.array([[1, 2, 3],
                   [4, 5, 6]])   # (2, 3)
arr_1d = np.array([10, 20, 30])  # (3,)->(1,3)

print(arr_2d + arr_1d)
# [[11 22 33]
#  [14 25 36]]  row broadcast down

col_vec = np.array([[100], [200]])  # (2,1)
print(arr_2d + col_vec)  # col across
When it fails

Shapes like (3, 4) + (2, 4) raise an error because 3 and 2 differ and neither is 1. When in doubt, print the .shape of each array before combining them.

Part 6 · The artificial neuron

output = activation(w · x + b)

A neuron multiplies input features by learned weights, sums them (a dot product), adds a bias, then passes the result through a non-linear activation such as ReLU, which is simply max(0, x).

inputs = np.array([1.5, 2.0, 3.5])    # 3 features
weights = np.array([0.2, 0.8, -0.5])  # 3 weights
bias = 2.0

def relu(x):
    return np.maximum(0, x)           # ReLU: max(0, x)

z = np.dot(weights, inputs) + bias    # weighted sum w.x + b
output = relu(z)                      # non-linear activation
You already know this

The whole neuron is one dot product, one broadcast add, and one element-wise activation, three vectorized ideas you already learned.

Part 7 · Layer of neurons

Stacking neurons with matrix multiply

A layer is many neurons at once. Stack the weight vectors into a matrix W of shape (n_neurons, n_features), then Z = W @ X + b computes every neuron's weighted sum in one matrix multiply, with the bias broadcast across neurons.

np.random.seed(42)
n_neurons, n_features = 4, 3
W = np.random.randn(n_neurons,
              n_features) * 0.5   # (4, 3)
b = np.random.randn(n_neurons)      # (4,)
X = np.array([1.0, 2.0, 3.0])   # (3,)

Z = W @ X + b   # (4,3)@(3,)->(4,)
A = relu(Z)     # element-wise
print(Z.shape, A)
This is nn.Linear

One matrix multiply replaces a loop over every neuron, and the bias add is pure broadcasting. This is literally what nn.Linear does inside PyTorch.

Part 8 · Vectorization vs loops

Proving the speedup

Build a large layer (1000 neurons, 500 features) and time the vectorized forward pass against an equivalent nested for loop.

import time
large_n_neurons, large_n_features = 1000, 500
W_large = np.random.randn(large_n_neurons, large_n_features)
b_large = np.random.randn(large_n_neurons)
X_large = np.random.randn(large_n_features)

start = time.time()
Z_vec = W_large @ X_large + b_large      # vectorized
vec_time = time.time() - start

start = time.time()
Z_loop = np.zeros(large_n_neurons)       # nested loops
for i in range(large_n_neurons):
    for j in range(large_n_features):
        Z_loop[i] += W_large[i, j] * X_large[j]
    Z_loop[i] += b_large[i]
loop_time = time.time() - start

print(f"Speedup: {loop_time / vec_time:.1f}x")
print(np.allclose(Z_vec, Z_loop))        # True

Part 8 · Vectorization vs loops

The result: not even close

The vectorized version runs in a fraction of a millisecond while the loop crawls, and np.allclose confirms both produce the same answer.

Where the speed comes from

Compiled C, SIMD parallelism, cache-friendly memory layout, and zero Python per-element overhead. It is the same algorithm, just run in a far faster place.

Part 9 · Real neural networks

The same math, at scale

A deep network just chains layers: H1 = f(W1 @ X + b1), then H2 = f(W2 @ H1 + b2), and so on. Real frameworks process a whole batch of samples at once by making X a 2D array.

batch_size, n_features = 3, 3
X_batch = np.random.randn(batch_size,
                        n_features)  # (3, 3)

# One forward pass for the whole batch
Z_batch = X_batch @ W.T + b
# (3,3)@(3,4)+(4,) -> (3,4)
A_batch = relu(Z_batch)
print(A_batch.shape)   # (3, 4)
# 3 samples, 4 neuron outputs
The leap to real ML

Batching is just adding a leading sample dimension and letting broadcasting handle the bias, so the jump from one sample to a mini-batch is a shape change, not a rewrite. PyTorch and TensorFlow add GPUs, autodiff, and batching on top of this exact math.

What you learned

  • Element-wise operations combine matching array elements with no loops, running in compiled C
  • Scalar and array broadcasting stretch smaller shapes to fit larger ones without copying memory
  • The three broadcasting rules (pad with 1s, equal-or-one compatibility, take the max size) predict every shape outcome
  • The dot product is a neuron's weighted sum; matrix multiplication does many at once as (m,n) @ (n,p) -> (m,p)
  • A neuron is f(w · x + b); a layer is f(W @ X + b)
  • Vectorized code is typically 10 to 1000 times faster than loops, and np.allclose confirms the match
  • A * B is element-wise while A @ B is matrix multiply; print .shape when debugging
The big idea

PyTorch and TensorFlow use these exact operations, adding GPUs, autodiff, and batching on top. You have now written the forward pass by hand.
