The Problem with Loops

Loop-Based Approach

  • Sequential execution
  • Python overhead per iteration
  • No parallelism

Vectorized Approach

  • Parallel execution (SIMD)
  • Optimized C implementation
  • Better cache utilization

Element-wise Operations

Operations applied to corresponding elements

  • Addition, subtraction, multiplication, division
  • Applied position by position
  • Arrays must have compatible shapes

The Artificial Neuron

Mimics biological neurons: weighted sum + activation

Forward Pass Math

y = f(w · x + b)

w = weight vector (learned parameters)

x = input vector (features)

b = bias (learned offset)

w · x = dot product (weighted sum)

f( ) = activation function (e.g., ReLU, sigmoid)

For ReLU: f(z) = max(0, z)

Single Neuron → Layer of Neurons

Single Neuron

y = f(w · x + b)

w: (n,)
x: (n,)
b: scalar
y: scalar

Layer of Neurons

Y = f(W @ x + b)

W: (m, n)
x: (n,)
b: (m,)
Y: (m,)

Each row of W is one neuron's weights
Matrix multiplication computes all neurons in parallel!

  • Layer of neurons: Y = f(W @ x + b)

  • GPUs accelerate matrix operations massively in parallel
