Why should you care?

Imagine thousands of LLM prompt-response records: which model answered, the task category, tokens used, and a user rating.

Before you can train anything, you need to know:

  • Which model performs best on which task
  • Whether any ratings are missing
  • Whether your task categories are balanced
Key insight

Doing this by hand or with raw Python loops is slow and error-prone. Pandas lets you load, inspect, filter, and aggregate that data in a few lines, which is the exact first step of every real AI research workflow.

Part 1 · From Spreadsheets to DataFrames

Pandas is the AI researcher's first tool

A DataFrame is like a spreadsheet you can control with code: rows, labeled columns, and a fast library for filtering and summarizing.

import pandas as pd

# The dataset tracks LLM interactions:
#   prompt_id, prompt_text, model, category,
#   temperature, tokens_used, rating, timestamp
df = pd.read_csv('data/llm_prompts.csv')
print(df.shape)  # (rows, columns), e.g. (100, 8)
Key insight

Before training, during experiments, and after deployment, the same skill (inspecting a DataFrame) is what tells you if your data is trustworthy.

Part 2 · Loading Data

Reading CSV and JSON

pd.read_csv() turns a file into a DataFrame and infers column types. For more control, declare types with dtype and parse dates with parse_dates. Pandas also reads JSON with pd.read_json().

from pathlib import Path

data_path = Path('data/llm_prompts.csv')

# Basic load
df = pd.read_csv(data_path)

# Load with explicit types and date parsing
df = pd.read_csv(
    data_path,
    dtype={'category': 'category'},
    parse_dates=['timestamp'],
)
Key insight

Getting types right at load time (categories, dates, ints) saves you from silent bugs later, for example dates that are really just strings you cannot sort.

Part 3 · DataFrame Anatomy & Series

Index, columns, values, and the Series

A DataFrame has three parts: an index (row labels), columns (feature names), and values (the 2D data). Selecting one column returns a Series, a single 1D labeled array.

# Inspect the parts
print(df.index)       # row labels
print(df.columns)     # column names

# A single column is a Series
model_series = df['model']
print(type(model_series))
# pandas.core.series.Series

# Data type of every column
print(df.dtypes)
Key insight

Understanding that a column is a Series explains why so many methods (.value_counts(), .mean(), .isna()) work directly on df['column'].

Part 4 · Inspection Methods

head, tail, info, describe, shape

.head(n) and .tail(n) eyeball rows. .info() reports row count, dtypes, and non-null counts. .describe() gives summary stats. .shape returns the (rows, columns) tuple.

df.head()          # first 5 rows
df.tail(3)         # last 3 rows

df.info()          # structure, dtypes, non-null counts
df.describe()      # numeric summary stats

rows, cols = df.shape
print(f"{rows} rows and {cols} columns")
Key insight

Always start with .head() to sanity-check the load, then .info() to spot missing values and wrong types before you trust any analysis.

Part 5 · Selecting & Filtering

Columns, positions, boolean indexing

Use df['col'] for a Series and df[['a','b']] for a DataFrame. Select rows by position with .iloc[]. The most powerful pattern is boolean indexing: pass a True/False condition to keep matching rows.

# Multiple columns -> a DataFrame
subset = df[['model', 'rating', 'tokens_used']]

# Rows by position
first_five = df.iloc[0:5]

# Boolean indexing: high-rated coding prompts
high_rated_coding = df[(df['category'] == 'coding') & (df['rating'] >= 4.7)]

# Keep rows matching several models
major_models = df[df['model'].isin(['gpt-4', 'claude-3'])]
Key insight

Combine conditions with & (AND), | (OR), and ~ (NOT), wrapping each in parentheses. Vectorized filtering replaces slow loops; for assignment use df.loc[condition, 'col'] to avoid the chained-indexing trap.

Part 6 · Finding Missing Data

Detecting and counting NaNs

.isna() returns a True/False grid marking every missing cell. Chaining .sum() counts missing values per column, and summing twice gives the total.

# Missing values per column
missing = df.isna().sum()
print(missing[missing > 0])

# Overall missing percentage
total_cells = df.shape[0] * df.shape[1]
missing_cells = df.isna().sum().sum()
print(f"Missing: {(missing_cells / total_cells) * 100:.2f}%")

# Inspect the rows that are missing a rating
missing_rating = df[df['rating'].isna()]
Key insight

In this dataset the rating column has missing values; later you must decide to drop those rows or fill them (for example with the median) before training.

Part 7 · Value Counts & Distributions

Categorical counts and histograms

.value_counts() tallies how often each category appears, revealing class imbalance. For numeric columns, a histogram shows the shape of the distribution.

# Categorical distribution
print(df['model'].value_counts())
print(df['category'].value_counts())

# As percentages
print(df['model'].value_counts(normalize=True) * 100)

# Numeric distribution (needs matplotlib)
df['tokens_used'].hist(bins=20)
Key insight

Class imbalance found here directly shapes ML strategy: it may call for oversampling, undersampling, or a weighted loss so the model is not biased toward the dominant category.

Part 8 · Grouping & Aggregation

groupby, agg, and pivot tables

groupby follows a split-apply-combine pattern: split rows by a category, apply a function, and combine the results. Pair it with .agg() to compute several stats at once.

# Average rating by model, with count and spread
model_stats = df.groupby('model')['rating'].agg(['mean', 'count', 'std'])
print(model_stats)

# Token usage by category
category_tokens = df.groupby('category')['tokens_used'].agg(['mean', 'min', 'max'])

# Compare model x category as a pivot table
pivot = df.groupby(['model', 'category'])['rating'].mean().unstack(fill_value=0)
Key insight

groupby is how you answer research questions like which model is best at coding, turning a raw log into a decision-ready comparison in one line.

Part 9 · The Lab

Dataset detective & the ML connection

The lab section of tutorial.ipynb walks the full workflow end to end, then ties each finding to a concrete ML preparation step.

# The core detective flow (in tutorial.ipynb)
df = pd.read_csv(data_path)
print(df.shape)
df.info()

Key insight

Inspection is not busywork: missing values, imbalance, and dtypes each map to a required ML step (imputation, resampling, encoding, scaling) before a model ever sees the data.

What you learned

  • A DataFrame is a labeled 2D table (index + columns + values); a Series is a single labeled column
  • pd.read_csv() and pd.read_json() load data; use dtype and parse_dates to get types right at load time
  • Start every analysis with .head(), .info(), .describe(), and .shape
  • Filter with boolean indexing and combine conditions using &, |, and ~ with parentheses
  • .isna().sum() reveals where data is missing so you can decide to drop or fill it
  • .value_counts() exposes class imbalance and histograms show numeric distributions
  • groupby + .agg() answer comparative questions, and .unstack() builds pivot tables
  • Every inspection finding maps to a real ML prep step: imputation, encoding, scaling, or resampling
The big idea

Understanding your data comes first. A few lines of Pandas turn a raw log into a trustworthy foundation for any model.
