An Image Is Just Numbers

To a computer, every image is a grid of numeric values representing color intensities.

Each pixel contains numerical values that define its color in the RGB color space.

© mui-group 2 / 15

The 3D Array Structure

Images are stored as 3-dimensional NumPy arrays with shape (Height, Width, Channels).

  • Height: Number of rows (pixels vertically)
  • Width: Number of columns (pixels horizontally)
  • Channels: Color components (3 for RGB)
© mui-group 3 / 15

Color Channels: R, G, B

Each pixel has three color channels: Red, Green, and Blue. Combined, they produce the full color spectrum.

OpenCV's BGR Quirk

OpenCV loads images in BGR format, not RGB. This is a historical quirk from early video standards.

⚠️ Important: Most libraries (matplotlib, PIL, PyTorch) expect RGB format. Always convert BGR → RGB after loading with OpenCV.
© mui-group 5 / 15

Loading Images with cv2.imread

OpenCV's cv2.imread() function loads an image file into a NumPy array.

import cv2 # Load image (returns BGR format) img = cv2.imread('photo.jpg') # Check the shape print(img.shape) # (480, 640, 3) # Height=480, Width=640, Channels=3 # Check data type print(img.dtype) # uint8 (0-255 range)
  • Returns a NumPy array of shape (H, W, 3)
  • Data type is uint8 (unsigned 8-bit integer)
  • Pixel values range from 0 to 255
  • Color format is BGR, not RGB
  • Returns None if file cannot be loaded
© mui-group 6 / 15

Resizing: Why 224×224?

Neural networks require fixed-size inputs. The standard size for ImageNet models is 224×224 pixels.

  • VGG, ResNet, MobileNet expect 224×224 input
  • Fixed size required for batch processing
  • Trade-off between detail and computation
  • Use cv2.resize(img, (224, 224))
© mui-group 7 / 15

Pixel Normalization

Convert pixel values from 0-255 integer range to 0-1 float range.

# Normalize pixel values img_normalized = img.astype(np.float32) / 255.0 # Check range print(img_normalized.min(), img_normalized.max()) # Output: 0.0 1.0
© mui-group 8 / 15

Why Normalize?

Normalization is essential for effective neural network training.

Benefits

  • Consistent scale across features
  • Faster gradient descent convergence
  • Prevents numerical instability
  • Required by pretrained models
  • Reduces internal covariate shift
© mui-group 9 / 15

Processing a Batch

The complete preprocessing pipeline for multiple images.

batch = [] for filename in image_files: img = cv2.imread(filename) img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB) img = cv2.resize(img, (224, 224)) img = img.astype(np.float32) / 255.0 batch.append(img) batch = np.stack(batch, axis=0)

Memory Considerations

Batch size directly impacts memory usage. Plan accordingly for GPU constraints.

# Calculate memory for one image (224×224×3, float32) pixels_per_image = 224 × 224 × 3 = 150,528 bytes_per_pixel = 4 # float32 memory_per_image = 150,528 × 4 = 602,112 bytes ≈ 0.57 MB
💡 Tip: Typical GPUs have 4-24 GB memory. Start with smaller batches (32-64) and increase if memory allows.

Key Takeaways

  • Images are 3D NumPy arrays with shape (H, W, C)
  • OpenCV loads images in BGR format — convert to RGB
  • Resize images to consistent dimensions for model input
  • Normalize pixel values from 0-255 to 0-1
  • Batch processing creates 4D arrays: (N, H, W, C)
  • Memory usage scales linearly with batch size
  • 224×224 is the standard size for ImageNet models

Connection to vision models: This preprocessing pipeline is the foundation for all computer vision tasks. ResNet, VGG, and other models expect exactly this input format.
