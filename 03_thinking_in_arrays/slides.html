Why should you care?

A language model cannot multiply the word "cat" by a weight matrix. It can only work with numbers laid out in memory.

Before any training happens, every token must become a numeric array. The way that array is stored decides everything:

  • Store it well and the CPU or GPU runs at hardware speed
  • Store it poorly and you are stuck at plain Python speed
Key insight

This module builds the bridge from text tokens to the numerical matrices that feed every neural network. The layout of those numbers in memory is not a detail, it is the difference between a model that trains and one that crawls.

Part 1 · Why Lists Fail

Why Python Lists Fail for AI

A Python list stores pointers to objects scattered across RAM. Each element carries object overhead, and every access risks a CPU cache miss. A NumPy array stores raw values in one contiguous block, so the CPU loads whole chunks into cache and runs vectorized SIMD operations.

© mui-group
6 / 18
Part 1 · Why Lists Fail

Same numbers, better layout

# Python list: references to objects,
# scattered across RAM (~28 bytes each)
#   [ref] -> object   cache misses

# NumPy array: raw values in one block
#   [val][val][val][val]  cache-friendly

import numpy as np
python_list = [1, 2, 3, 4, 5]
arr = np.array(python_list)   # contiguous
print(type(arr))   # <class 'numpy.ndarray'>
Key insight

The speedup is not magic, it comes from memory layout. Contiguous raw values plus vectorization and SIMD are exactly what GPUs and training loops depend on.

Part 2 · Creating Arrays

Creating NumPy Arrays

You rarely build arrays element by element. You create them from lists, or with factory functions. Each array has a dtype, its single homogeneous element type, that NumPy infers or you specify.

import numpy as np

arr_from_list = np.array([1, 2, 3, 4, 5])   # from a Python list
print(arr_from_list.dtype)                  # int64

zeros = np.zeros(5)                         # [0. 0. 0. 0. 0.]
ones = np.ones(5)                           # [1. 1. 1. 1. 1.]
range_arr = np.arange(0, 10, 2)             # [0 2 4 6 8]  (start, stop, step)
linspace_arr = np.linspace(0, 1, 5)         # [0.   0.25 0.5  0.75 1.  ]
Why it matters

np.zeros is how you pre-allocate the empty one-hot matrix you will fill in later, a pattern that shows up directly in the lab.

Part 3 · The .shape Property

The .shape Property

The .shape attribute is a tuple giving the size of each dimension. Reading shapes fluently is the single most important debugging skill, because most array errors are really shape mismatches.

import numpy as np

arr_1d = np.array([1, 2, 3, 4, 5])
print(arr_1d.shape)   # (5,)      1D, 5 elements

arr_2d = np.array([[1, 2, 3], [4, 5, 6]])
print(arr_2d.shape)   # (2, 3)    2 rows x 3 cols

arr_3d = np.zeros((2, 3, 4))
print(arr_3d.shape)   # (2, 3, 4) 2 matrices of 3x4
Scales to real data

The same shape grammar scales straight to AI data: (batch, height, width, channels) for images and (sequence_length, features) for sequences.

Part 4 · The Shape Bug

Rank-1 vs Rank-2 Arrays

Three arrays can all hold five numbers yet behave differently. Many NumPy operations return the ambiguous (n,) form, but matrix operations expect an explicit 2D shape. That mismatch is a classic source of cryptic bugs.

import numpy as np
rank1 = np.array([1, 2, 3, 4, 5])
print(rank1.shape)   # (5,)  the trap

column = np.array([[1], [2], [3], [4], [5]])
print(column.shape)  # (5, 1) 2D column

row = np.array([[1, 2, 3, 4, 5]])
print(row.shape)     # (1, 5) 2D row

# Fix: -1 means 'infer this axis'
print(rank1.reshape(-1, 1).shape)  # (5, 1)
print(rank1.reshape(1, -1).shape)  # (1, 5)
Rule of thumb

For anything doing matrix math, never trust a bare (n,) shape. Force it to (n, 1) or (1, n) so operations behave predictably.

Part 5 · Reshaping

Reshaping Arrays

Reshaping reinterprets the same contiguous data under a new shape without copying it, as long as the total element count stays constant. Pass -1 to let NumPy infer one dimension.

import numpy as np
original = np.arange(12)      # shape (12,)

print(original.reshape(3, 4))  # 3 x 4
print(original.reshape(4, 3))  # 4 x 3

auto = original.reshape(2, -1) # -1 infers axis
print(auto.shape)             # (2, 6)

# Total must match: 12 = 3*4 = 2*6,
# but 5*3 = 15 raises an error.

Part 6 · One-Hot Encoding

One-Hot Encoding a Sentence

One-hot encoding turns discrete tokens into numeric vectors. Build a vocabulary that maps each unique word to an index. Each word becomes a vector of all zeros with a single 1 at its index, and a sentence becomes a matrix: one row per word, one column per vocabulary position.

© mui-group

Part 6 · One-Hot Encoding

Building the matrix

Pre-allocate a block of zeros with np.zeros, then flip a single 1 per row. This gives every word a unique, orthogonal representation with no accidental ordering.

import numpy as np

def build_vocabulary(text):
    unique_words = sorted(set(text.lower().split()))
    return {word: idx for idx, word in enumerate(unique_words)}

def create_onehot_matrix(text, vocab):
    words = text.lower().split()
    onehot_matrix = np.zeros((len(words), len(vocab)))  # pre-allocate zeros
    for i, word in enumerate(words):
        idx = vocab.get(word, -1)
        if idx >= 0:
            onehot_matrix[i, idx] = 1                    # set the single 1
    return onehot_matrix

text = "the cat sat on the mat"
vocab = build_vocabulary(text)
matrix = create_onehot_matrix(text, vocab)
print(matrix.shape)   # (6, 5)  6 words x 5 vocab size
The hand-off point

This is the exact hand-off from a tokenizer to a model: tokens in, a numeric matrix out, ready to be fed forward.

Part 7 · The Sparsity Problem

The Sparsity Problem

One-hot works but scales badly. With a realistic 50,000 word vocabulary and a 20 word sentence, the matrix holds a million values yet only 20 are nonzero. That is about 99.96 percent zeros.

vocab_size = 50000
sentence_length = 20

total_elements = sentence_length * vocab_size
# 1,000,000
nonzero_elements = sentence_length
# 20 (one per row)

sparsity = (1 - nonzero_elements / total_elements) * 100
print(f"Sparsity: {sparsity:.4f}%")
# Sparsity: 99.9600%
Two problems, not one

It wastes memory and computation, and worse, every word is equally distant from every other. There is no notion that cat is closer to dog than to table.

Part 8 · Connection to LLMs

From One-Hot to LLM Embeddings

One-hot vectors are stage one of the real LLM pipeline. An embedding layer maps each sparse one-hot vector to a dense vector of a few hundred continuous values that carry meaning, so similar words end up with similar vectors. Transformers then make those embeddings contextual.

The on-ramp

Everything in this module is the on-ramp: text becomes tokens, tokens become one-hot arrays, and arrays become the embeddings that the next module builds.

Part 8 · Connection to LLMs

The pipeline, stage by stage

# The LLM input pipeline, stage by stage:
# 1. Tokenizer:        text            -> tokens
# 2. One-hot:          'cat'           -> [0, 0, 1, 0, ..., 0]  (50,000 dims, sparse)
# 3. Embedding layer:  one-hot         -> [0.24, -0.81, 0.53, ...] (768 dims, dense)
# 4. Transformer:      dense + context -> contextual vector
# 5. Neural network:   learns semantic relationships from the dense vectors
Why embeddings win

Sparsity plus zero semantics is precisely the pain that motivates dense embeddings: compress 50,000 dimensions down to a few hundred meaningful ones, where the same word can even shift meaning with context.

What you learned

  • NumPy arrays store raw values in contiguous memory, roughly 50 to 100x faster than lists for numerical work
  • The .shape tuple reveals dimensions: (n,) vector, (rows, cols) matrix, (depth, rows, cols) tensor
  • Rank matters: (n,) is not (n, 1) or (1, n), and reshape with -1 fixes the shape bug
  • Reshaping reinterprets data without copying, as long as the total element count stays constant
  • One-hot encoding converts discrete tokens into sparse, orthogonal binary vectors, one row per word
  • One-hot is about 99.96 percent zeros and carries no semantics, which is why dense embeddings come next
  • The full input pipeline is Text to Tokens to One-hot to Embeddings to Neural Net
The big idea

How numbers are laid out in memory decides whether an AI system runs at Python speed or hardware speed. Arrays are that layout.
