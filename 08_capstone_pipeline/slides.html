Why should you care?

Real machine learning almost never starts with clean tensors. It starts with a folder of images and a spreadsheet of captions.

Every model, from a ResNet classifier to a CLIP-style multimodal model, needs that raw data:

  • Cleaned and validated so bad rows never reach training
  • Batched so the GPU is fed efficiently
  • Shaped correctly before a single gradient can flow
Key insight

The data pipeline is the unglamorous but load-bearing layer that decides whether training even runs. Building one end to end is the capstone skill of this course.

Part 2 · The Manifest Pattern

Link Metadata to Images with a CSV

Instead of encoding labels into file names, a manifest CSV keeps a separate row per sample that points to an image file and carries its caption, category, and split.

# data/manifest.csv
image_filename,caption,category,split
galaxy_001.png,Spiral galaxy with
  bright core,astronomy,train
cell_001.png,Human red blood cells
  under microscope,biology,train
landscape_001.png,Mountain range with
  snow peaks,geography,train
# 20 samples across 5 categories,
# split 16 train / 4 test

Why this works:

  • Metadata stays independent of the pixels
  • Train/test filtering becomes trivial
  • Works with existing images without renaming anything
Key insight

Every large-scale dataset (COCO, ImageNet, LAION) ships as a manifest plus a blob of files. Learning the pattern here means you can point this same loader at a real dataset by only swapping the CSV.

Generate the Dataset Before You Load It

img = np.zeros((h, w, 3), dtype=np.uint8)
center = (h // 2, w // 2)
cv2.circle(img, center, 40 + index * 5, (255, 240, 200), -1)  # galaxy core
cv2.imwrite(str(output_path), img)
Common error

Skipping the generation step is the most common mistake, because the loader validates that every file actually exists. If you see "images not found", you forgot to run create_sample_data.py.

Key insight

Synthetic data lets you develop and test a pipeline before spending money on real data or GPUs. The same OpenCV calls that draw these images are the ones you will later use for data augmentation.

Part 4 · Loading and Cleaning

Read, Deduplicate, and Validate

The loader's __init__ reads the CSV with Pandas and then applies the Module 6 cleaning steps so that every remaining row is a usable sample.

df = pd.read_csv(self.manifest_path)
initial_count = len(df)

df = df.drop_duplicates(subset=['image_filename'])
df = df.dropna(subset=['image_filename', 'caption', 'category', 'split'])
df = df[df['split'] == self.split].copy()

df['caption'] = df['caption'].str.strip()
df['caption'] = df['caption'].str.replace(r'\s+', ' ', regex=True)

valid_mask = df['image_filename'].apply(lambda x: (self.image_dir / x).exists())
df = df[valid_mask].copy()
df = df.reset_index(drop=True)
Key insight

Bad data produces bad models. Doing validation once at load time (including the on-disk file check) means the training loop never crashes halfway through an epoch on a missing image.

Part 5 · Preprocessing Images

From File on Disk to Normalized Array

A private helper loads one image at a time, resizes it, converts from OpenCV's default BGR ordering to RGB, and optionally normalizes pixels into the [0, 1] range.

img = cv2.imread(str(image_path))          # loads as BGR
if img is None:
    return None

img = cv2.resize(img, (self.target_size[1], self.target_size[0]))
img = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)  # BGR to RGB

if self.normalize:
    img = img.astype(np.float32) / 255.0     # scale to [0, 1]

return img
Two gotchas

OpenCV reads BGR (not RGB), and networks expect normalized floats (not 0 to 255 uint8). Getting either wrong trains a model that silently underperforms rather than erroring out.

Part 6 · The Generator Pattern

Yield One Batch at a Time

The __iter__ method makes the loader iterable using a generator. Because it yields instead of returning a list, only one batch lives in memory at a time.

indices = np.arange(len(self.df))
if self.shuffle:
    np.random.shuffle(indices)

for start_idx in range(0, len(self.df), self.batch_size):
    end_idx = min(start_idx + self.batch_size, len(self.df))
    batch_indices = indices[start_idx:end_idx]
    batch_df = self.df.iloc[batch_indices]

    image_list, text_list = [], []
    for idx, row in batch_df.iterrows():
        img = self._load_and_preprocess_image(self.image_dir / row['image_filename'])
        if img is not None:
            image_list.append(img)
            text_list.append(row['caption'])
    if not image_list:
        continue
    yield {...}
Key insight

The generator pattern is what lets a data loader handle datasets far larger than RAM. This is the exact mechanism PyTorch and TensorFlow rely on for streaming data during training.

Part 7 · Stacking with NumPy

Build the Batch and Read Its Statistics

Once a list of preprocessed images is collected, np.stack combines them into a single (B, H, W, C) array, and the loader attaches a metadata dictionary.

image_batch = np.stack(image_list, axis=0)   # -> (B, H, W, C)

yield {
    'image_batch': image_batch,               # float32 in [0, 1]
    'text_batch': text_list,                  # List[str] of length B
    'metadata': {
        'batch_size': len(image_list),
        'categories': batch_df.iloc[valid_indices]['category'].tolist(),
        'filenames': batch_df.iloc[valid_indices]['image_filename'].tolist(),
        'split': self.split,
    },
}

# Later, inspecting a batch:
print(f"Shape: {images.shape}, range: [{images.min():.3f}, {images.max():.3f}]")
print(f"Mean: {images.mean():.3f}, Std: {images.std():.3f}")
Key insight

Checking a batch's shape, dtype, and value range before feeding a model is the fastest way to catch preprocessing bugs. A mean near 0.5 and a [0, 1] range confirm normalization worked.

Part 8 · Timing and Splits

Measure the Pipeline, Train vs Test

The main function runs the loader over the training split with shuffling on, times the full pass, then runs the test split with shuffle off, since evaluation data should be iterated in a fixed order.

train_loader = MultimodalDataLoader(
    manifest_path=manifest_path, image_dir=image_dir,
    batch_size=4, target_size=(224, 224),
    split="train", shuffle=True, normalize=True,
)

start_time = time.time()
batch_count = total_samples = 0
for batch_idx, batch in enumerate(train_loader):
    batch_count += 1
    total_samples += len(batch['image_batch'])
elapsed = time.time() - start_time
print(f"Samples per second: {total_samples/elapsed:.1f}")

# Test split iterates deterministically:
test_loader = MultimodalDataLoader(..., split="test", shuffle=False)
Key insight

Shuffle training data to break ordering bias, but never shuffle evaluation data so metrics are reproducible. Timing the loader early tells you whether data loading (not the GPU) will be your training bottleneck.

Part 9 · Connecting to Frameworks

NumPy Out, Tensors In

The pipeline's NumPy output plugs directly into both major frameworks. Our output is channel-last, so TensorFlow is a drop-in and PyTorch needs one permute.

# PyTorch: needs channel-first ordering
import torch
image_tensor = torch.from_numpy(batch['image_batch'])  # zero-copy
image_tensor = image_tensor.permute(0, 3, 1, 2)         # (B,H,W,C) -> (B,C,H,W)
output = model(image_tensor)

# TensorFlow: keeps channel-last (matches our output)
import tensorflow as tf
image_tensor = tf.convert_to_tensor(batch['image_batch'])  # already (B,H,W,C)
output = model(image_tensor)
Most common integration bug

Channel ordering: PyTorch wants (B, C, H, W) while TensorFlow wants (B, H, W, C). Our pipeline emits channel-last, so TensorFlow is a drop-in and PyTorch needs one permute.

What you learned

  • A data pipeline is a first-class engineering artifact: model it as a reusable class with clear configuration and output contracts
  • The manifest pattern separates metadata from files, making filtering, splits, and new datasets easy without renaming anything
  • Clean and validate data once at load time, including the on-disk file existence check, so training never crashes mid-epoch
  • OpenCV loads BGR and returns 0 to 255 uint8: always convert to RGB and normalize to [0, 1] float32 before a model sees the data

What you learned

  • The generator pattern yields one batch at a time, giving memory efficiency that scales to datasets larger than RAM
  • np.stack turns a list of images into a (B, H, W, C) batch, and NumPy reductions give you instant sanity checks on shape and range
  • Shuffle training data but never evaluation data, and time the loader to know if data loading is your bottleneck
  • PyTorch is channel-first (B, C, H, W) and TensorFlow is channel-last (B, H, W, C): our pipeline emits channel-last
The big idea

The data pipeline is where raw reality meets the model. Master it, and you can point a neural network at almost any dataset in the world.
